组件化

React 都是关于构建可复用的组件。通过 React 你唯一要做的事情就是构建组件。 得益于其良好的封装性，组件使代码复用、测试和关注分离（separation of concerns）更加简单。

虚拟DOM

React不操作 DOM，除非不得不操作 DOM 。它用一种更快的内置仿造的 DOM 来操作差异，为你计算出效率最高的 DOM 改变。只会更新变化的部分。

Redux

Redux 是什么？

Redux 是应用状态管理服务

它虽然受到了 Flux 很深的影响，但是核心概念更加简单，就是 Map/Reduce 中的 Reduce

Store介绍

一个应用只有一个Store

保存应用的状态，提供一些方法给组件调用

Action介绍

Actions 是把数据从应用传到 store 的有效载荷。它是 store 数据的唯一来源

Action用法

普通的JavaScript 对象，必须有type 属性；

例如 { type: 'ADD_TO_DO', text: 'Learn Redux' }

通过store.dispatch(action) 把 action 传到 store

State介绍

普通的JavaScript 对象来定义你的应用的状态，所有的状态都在这个对象中管理。

例如： { todos: [], filter: 'SHOW_ALL' }

Reducer介绍

Action 只是描述了有事情发生了这一事实，并没有指明应用如何更新 state。这是 reducer 要做的事情。

Action就像leader，告诉我们应该做哪些事，并且给我们提供‘资源（就是上面说的数据）’，真正干活的是苦逼的Reducer。。

普通的JavaScript 函数，state和 action作为参数传入reducer 函数，经过相应的计算，返回新的state 对象

Reducer用法

一个普通的JavaScript 函数，state和 action作为参数传入reducer 函数，经过相应的计算，返回新的state 对象
